Bat-first pools (only used in bat_first_pool_xi mode)

Auto-fill sorts ticked players: bowler/AR → P2, else P1. Every ticked player must be in exactly one pool. Lineups are only 6+5 (Seg A) or 5+6 (Seg B). Set segment weights for full-pool runs and separately for reduced-pool runs (after exclusions); values are normalized to 100%. Generation uses three passes for same-XI caps.

2nd innings pools (only used in second_innings_pool_5 mode)

Auto-fill puts every ticked player from the chasing team into Top (WK + bat slot ≤3) or Rest (everyone else on that team, including bowlers and AR). First-innings P2 is unchanged: that side’s BOWL + AR only. Trim lists manually. Generation uses two scenarios: chase-middle vs first-innings-bowl (see % and combo rows below); teams are merged in alternating order.

Captain / vice-captain pools (per scenario)

Players from chase Top ∪ Rest ∪ P2. Tick C and/or VC per scenario (optional trim). Chase-middle: C from Top/Rest, VC from Rest/P2. Bowl: C from P2, VC from Top/Rest/P2. Auto-fill applies pool rules; generation uses pool rules if no ticks.

Split pools (only used in split_pool_p1p2p3 mode)

Per franchise: P1 tops · P2 middle bat · P3 bowlers + bowling AR. Auto-fill assigns ticked players. Generation uses three passes for same-XI caps (Dup pass 1/2/3). Set C1–C4 % for how many teams use each profile (values normalized to 100%; default 25 each). Teams are built in round-robin order C1→C2→C3→C4→C1… (not all C1 then all C2). C/VC pools below use the same four profile ids (not α/β match-outcome cells). Optional full pool % + exclusions narrow P1/P2/P3 after the first half of teams.

Captain / vice-captain pools (per scenario)

Each scenario lists suggested C/VC picks by role. Use Add other ticked players for extras. Hints use franchise codes (e.g. CSK, MI) and who bats first vs chases for that cell.
